  <action>
In the `<script>` block, add the sort functions BEFORE the `updateConstraintCount` function (so they are defined before any event handlers that might call them). Add the following JS block:

```javascript
  var sortState = { col: 3, asc: true };  // default: Player A-Z

  function sortTable(colIndex) {
    var asc = (sortState.col === colIndex) ? !sortState.asc : false;  // first click = descending
    sortState = { col: colIndex, asc: asc };
    var tbody = document.querySelector("#player-pool-section table tbody");
    if (!tbody) return;
    var rows = Array.from(tbody.querySelectorAll("tr"));
    rows.sort(function(a, b) {
      var aVal = a.cells[colIndex] ? a.cells[colIndex].dataset.sort : "";
      var bVal = b.cells[colIndex] ? b.cells[colIndex].dataset.sort : "";
      var aNum = parseFloat(aVal);
      var bNum = parseFloat(bVal);
      var cmp = (!isNaN(aNum) && !isNaN(bNum))
        ? aNum - bNum
        : (aVal || "").localeCompare(bVal || "");
      return asc ? cmp : -cmp;
    });
    rows.forEach(function(r) { tbody.appendChild(r); });
    updateSortIndicators(colIndex, asc);
  }

  function updateSortIndicators(activeCol, asc) {
    document.querySelectorAll("#player-pool-section table thead th").forEach(function(th, i) {
      th.textContent = th.textContent.replace(/\s*[▲▼]$/, "");
      if (i === activeCol) {
        th.textContent += " " + (asc ? "▲" : "▼");
      }
    });
  }
```

After all function definitions and after the page-load initialization block (the updateConstraintCount() call added in Plan 01), add one more call to set the initial sort indicator:

```javascript
  updateSortIndicators(3, true);
```

This shows ▲ on the Player column header immediately on page load (before any user clicks), matching the default sort state (Player A-Z, which the server already renders in that order).

**Critical notes (from RESEARCH.md pitfall #1):**
- `th.textContent` setter is safe to use here. The `onclick="sortTable(N)"` is an HTML attribute — it is NOT affected by changes to textContent. Only child text nodes are replaced.
- The regex `/\s*[▲▼]$/` strips any trailing indicator from previous sort clicks before adding the new one.
- `tbody.appendChild(r)` moves existing rows (no clones) — checkbox name/value attributes are fully preserved, so form submission still works after sorting.

**Guard:** `if (!tbody) return;` in sortTable() handles the case where the player pool table is not in the DOM (e.g., on GET with no results). `updateSortIndicators(3, true)` at page load is also safe — if no `#player-pool-section` exists, querySelectorAll returns empty NodeList and forEach is a no-op.
